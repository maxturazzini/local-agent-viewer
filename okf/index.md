# LAV Interaction Metadata Bundle

This bundle documents, in [OKF](https://github.com/GoogleCloudPlatform/knowledge-catalog) format, the metadata schema attached to every **interaction** stored in the LocalAgentViewer (LAV) knowledge base.

An *interaction* is a single agent session — a conversation with Claude Code, ChatGPT, Codex CLI, or Cowork Desktop. LAV normalizes all of them into one schema and indexes them into a Qdrant vector collection. Each interaction carries two layers of metadata:

1. **Provenance** — captured deterministically at index time (who, where, when, how big).
2. **Semantic enrichment** — generated by an LLM classifier (what the interaction was *about* and how sensitive it is).

This bundle is the source-of-truth documentation for both layers. The fields themselves are defined in code at:

- `lav/classifiers/openai_classifier.py` — the JSON schema (`CLASSIFICATION_SCHEMA`) and sanitization rules
- `lav/classifiers/openai_strict.py` — the classifier system prompt (field semantics + decision rules)
- `lav/config.py` — the controlled vocabularies (`CLASSIFICATIONS`, `SENSITIVITIES`)
- `lav/qdrant/kb_indexer.py` — the full Qdrant payload assembled per interaction

## Quick Reference

A single interaction payload, as stored in Qdrant:

```json
{
  "session_id": "a4d7f0de-a96d-450a-9c3d-bb7c0f0810fb",
  "project": "miniMe",
  "user": "maxturazzini",
  "host": "macChia",
  "source": "claude_code",
  "timestamp": "2026-02-23T15:51:04.985Z",
  "indexed_at": "2026-04-05T00:43:19.232269",
  "summary": "...",
  "abstract": "...",
  "process": "blog publishing preparation",
  "topics": ["blog", "content management"],
  "people": [],
  "clients": [],
  "classification": "support",
  "data_sensitivity": "internal",
  "sensitive_data_types": ["internal_architecture"],
  "tags": [],
  "message_count": 165,
  "tools_used": []
}
```

> **Note on drift**: live payloads observed in Qdrant may lag the code. The `process` field exists in the classifier schema but is absent from some older indexed records; the `kb_indexer.py` docstring lists an outdated subset of `classification` values. The code (`config.py`) is authoritative — see [schema.md](schema.md) for the reconciled list.
